Strategic Recommendations for Communication
The "One-Way Valve" Rule: Tell your agent that Intent and Actual nodes are strictly children of a TimeChunk. This prevents the agent from creating floating events that aren't anchored to your timeline.
Schema Enforcement: Instruct the agent to run a "Schema Check" query before every write operation. It should check if the labels it’s about to use exist in your Mermaid map.
Pillar Overlap: Explicitly state: "A single TimeChunk can support multiple Pillars. Create multiple :CLASSIFIED_AS relationships from a single Event node when necessary."
